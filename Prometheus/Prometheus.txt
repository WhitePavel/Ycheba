Monitoring - это информация о части вашей системы в определённое время
 * CPU Utilization
 * Disk Space
 * Количество файлов в директории
 * Количество ошибок http 404 за последние 5 минут
 * Количество запросов http GEt за последние 5 минут
 
Для чего нужен Monitoring
 * Нужен для помощи в Debugging
 * Можно видеть состояние системы до, во время, после проблемы
 * Нужен что бы видеть тренд проблемы и приближающиеся проблемы
 * Нужен для автоматизации
------------------------------------------------------------------------
Monitoring System Types

PUSH system
* На клиент серверах установлен Monitoring Agent
* Этот Monitoring Agent собирает данные о системе
* Этот Monitoring Agent посылает данные на Monitoring Server
(Prometheus работает по системе PULL)
PULL System
* На клиент серверах установлен Monitoring Agent
* Этот Monitoring Agent собирает данные о системе 
* Этот Monitoring никуда ничего не посылает
* Этот Monitoring делает данные доступными на месте доступа
* А вот Monitoring Server собирает доступные данные из мест доступа
------------------------------------------------------------------------------

Monitoring vs Observability

Monitoring
 * это процесс сбора данные и создание отчётов по различным показателям,определяющим работоспособность системы
 * Мониторинг это сбор данных отдельных компонентов

Observability 
* Исследовательскй процесс, он изучает взаимодействие комапонентов распределённой системы и данные собранные в ходе мониторинга что бы найти основную причину проблем (root cause)
* Ищет какой компонент повлиял на что

Prometheus Metrics 

* После того как вы устанавливаете и запускаете Exporter, все метрики которые он мониторит и собирает будут доступны как Web страничка под определённым HTTP портом
* Нарпимер Node Exporter:
 * Если Private IP Вашего сервера 10.0.0.1.
 * Node Exporter публикует все метрики на HTTP Port #9100
 * По адресу: http://10.0.0.1:9100 вы увидите информацию о Exporter
 * По адресу:http://10.0.0.1.:9100/metrics вы увидите сами метрики

Prometheus Metric Types 
   Counter 
* метрика которая может только увеличиваться
* можно обнулить только перезапустив сервис который мониторится 
   Gauge 
* Метрика которая может увеличиваться и уменьшаться
   Histogram
* показывает Histogram - распределение велечин метрики по группам
   Summary
* показывает Percetile /Quantile из Histogram
