---------------------просмотр нодов/подов-------
kubectl get pods - просмотр какие поды бегут сейчас 
kubectl get nodes - просмотр нодов

-----------------ЗАПУСК ПОДА-------------

kubectl apply -f ***** - поднятие через yaml файл И ТАК ЖЕ ОБНОВЛЕНИЕ НА НА НОВЫЙ ИМИДЖ,
после нужн основа делать port-forward для проверки, НО МЕНЯТЬ МОЖНО ТОЛЬКО ИМЭДЖ,ОСТАЛЬНОЕ СМЕНЕ НЕ ПОДЛЕЖИТ
***** - название файла


kubectl port-forward hello 1488:80 - перекидывание порта нода,бегущего в поде с пода на локальный компьютер.
***** - название пода
$$$:%%% - порты(наш:контейнер)

kubectl run hello -- image=lains02/k8sphp:latest --port=80 -открытие пода с портом 80
*kubectl run* - запустить что-то
*hello* - название этого чегото
*--image=$$$$$$:%%%%* - выбираем image с докерхаб
*--port=80*
схематически получается следующее
kubectl run (НАЗВАНИЕ) (--докеримэдж) (--открытие порта)
