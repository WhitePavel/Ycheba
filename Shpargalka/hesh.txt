git хранит ифнормацию о результате хэширования
хэш => ифгрмация о коммите
если занешь хэш можно узнать всё остальное.

при работе с git  хэш можно укахывать в качестве параметра,что бы укзать с 
каким коммитом нужно произвести действие.

Все хэши и таблицу gt хранит в скрытой папке .git в репозитории проекта.

Если хэш получить дважды для одного и того же файла
то результат будет одинаковый. 
Если меняется хоть один символ то кэш измениться очень сильно.

Получение сокращённого лога
git log --oneline

Поулчение обычного лога
git log

Выход из просмотра логов клавишей Q

Когда делаем коммит git обновляет refs/heads/master в котором находится хэш последнего коммита
HEAD обновляется так как ссылается на refs/hads/master

Если нужно передать последний коммит то вместо хэша можно написать слово HEAD. Гит поймёт ч то нужен последний коммит
