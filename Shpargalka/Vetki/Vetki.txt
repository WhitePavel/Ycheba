Что бы просмотреть ветки которые есть в текущем репозитории нужно ввести команду
git branch

для создания новой ветки в репозитории нужно ввести команду 
git branch ****
****- название новой ветки репозитория

git checkout -b - команда нужна что бы создать новую ветку и сразу в ней работать

git checkout **** - переход в другую ветку 
**** - название ветки в которую нужно перейти


например можно узнавать разницу между коммитами с помощью команды git diff с флагом ~ при
указании нумерации он позволяет откатиться на N-ое количество комитов назад.

процесс слияния веток происходит при попомощи команды git merge ****
****- название ветки которую хочешь слиять с той веткой в которой щас находишься.

git branch -d/-D - команда удалаяющая ветку в первом случае оана удалит её
только после объединения,во втором уже в любом случае.


состтояние fast-forward - это то состояние когда при слиянии двух веток одна
переходит в дургую последовательно добваляя в неё коммиты.

данную команду можно отключить при помощи 
git merge --no-ff ****
*** - на один раз
git config [--global] merge.ff false

при отключении этого режима гит будет создавать коммит слияния или мёрж-коммит
при данном раскладе будет видно какая ветка принесла какие коммиты.

git push --force - форсированный пуш который удаляет коммит в осноной ветке с
ошибкой и позволяет обойти ошибку.
rebase - перевод ветки из состояние no fast forward в fast forward
перебахирование своей ветки так что бы она росла от предыдущего коммита.


всегда перед пушем в fast forward лучше запуллить основную ветку дабы было
меньше возможных конфликтов слияния

git push --set-upstream - это полное название ветки -u
git push -u
а HEAD - синоним текущей ветки
